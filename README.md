# Websites List App (WLA)

### 🔗 [View Application](https://dyaroman.github.io/wla-react/)

**Websites List App (WLA)** is a powerful, interactive index and management tool designed to track, filter, and analyze large collections of web applications.

> ⚠️ **Data Privacy Note:** All production data has been replaced with realistic mock data to comply with non-disclosure agreements (NDAs) and prevent legal issues with the data owner.

**v2.4.0** — React 19, Redux Toolkit, Vite 8, Sass

---

## 🚀 Why This Project Exists

This application was created to bridge the gap between technical infrastructure and business management, solving three core issues:

### 1. Feature & Campaign Auditing (Management)

Company management frequently needs to audit which websites have (or lack) specific configurations, trackers, forms, or `campaignIds`. WLA provides an instant, searchable overview of the entire web ecosystem.

### 2. Boilerplate & Donor Selection (Development)

When launching a new website, developers often look for the best existing "donor" site to duplicate or use as a baseline. WLA allows you to filter by features to find the perfect starting point.

### 3. Feature Build Indexing (CI/CD & QA)

During active development, feature branches often deploy multiple test environments. WLA serves as a dynamic index page where QA teams and stakeholders can:

- See exactly which websites belong to a specific feature build.
- Sort and filter environments based on their needs.
- Use a built-in **tracking checkbox** system to mark which sites have already been reviewed and which ones still need attention.

## 🎯 Educational Goals

Beyond solving real business problems, this repository serves as a personal sandbox to master modern web development practices. The focus is on implementing:

- Production-grade frontend frameworks and state management.
- Diverse libraries for handling data grids, filtering, and sorting.
- Multiple architectural patterns to compare different implementation strategies.
